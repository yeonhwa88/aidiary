추상 클래스란 : 추상 클래스는 하나 이상의 추상 메서드를 포함하는 클래스이며, 인스턴스를 직접 생성할 수 없다.
사용 목적 : 공통 기능을 제공하면서도 일부 기능은 하위 클래스에서 구현하도록 강제할 때 사용된다.
abstract 키워드 : 클래스나 메서드 앞에 abstract 키워드를 붙여 추상 클래스를 정의한다.
추상 메서드 : 메서드 본체가 없는 메서드로, 자식 클래스에서 반드시 구현해야 한다.
일부 구현 가능 : 추상 클래스는 추상 메서드뿐만 아니라 일반 메서드도 포함할 수 있어 일부 구현을 제공할 수 있다.
상속을 통한 구현 : 추상 클래스를 상속받은 자식 클래스는 추상 메서드를 반드시 구현해야 한다.
인스턴스화 불가 : 추상 클래스는 직접 인스턴스를 생성할 수 없으며, 자식 클래스를 통해서만 객체를 만들 수 있다.
인터페이스와 차이점 : 추상 클래스는 필드와 구현된 메서드를 가질 수 있지만, 인터페이스는 오직 메서드 시그니처만 정의할 수 있다.
