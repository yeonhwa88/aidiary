접근 제한자란? 
클래스, 변수, 메소드 등의 접근 권한을 제어하는 키워드로, 외부에서 어떤 요소에 접근할 수 있는지 결정.
public: 
모든 클래스에서 접근 가능. 가장 넓은 접근 범위를 가진다.
private: 
해당 클래스 내부에서만 접근 가능하며, 외부 클래스나 자식 클래스에서 접근할 수 없다.
protected: 
같은 패키지 내의 클래스나 자식 클래스에서 접근 가능하며, 상속받은 클래스에서도 접근 가능하다.
(default): 
아무 접근 제한자를 명시하지 않으면 default가 적용되며, 같은 패키지 내에서만 접근 가능.
public vs private: 
public은 외부 접근이 가능해 자유로운 사용이 가능하지만, private은 외부 접근을 제한해 데이터 은닉성을 강화.
protected vs default: 
protected는 상속 관계에서 유리하며, default는 같은 패키지 내에서만 유효.
getter/setter: 
private 필드를 외부에서 간접적으로 접근하게 할 때 사용하며, 메소드를 통해 값을 읽거나 수정할 수 있다.
