1. Web이란 ?
(1) Web이란 사전적인 의미로 거미줄이란 뜻으로 하나의 사이트나 또 다른 사이트와의 관계가 거미집처러 얽혀 있는 것에 비유.
	- WWW(wolrd wide web)의 줄임말
	- 정보 공유를 하는데 주 목적이 있다.
(2) Web이란 1개 이상의 사이트가 연결되어 있는 인터넷 서비스의 한가지 형태 ( 80 : 기본적인 포트번호 )
    - 프로토콜 : 네트워크상에서 약속한 통신규약
	-- Http : HyperText Transfer Protocol 정보를 가져와 브라우저 화면상에서 보임.
	-- FTP :  File Transfer Protocol. 파일을 다운
	-- SMTP, POP : 메일 송수신 protocol
	-- DHCP : Dynamic Host Configuration Protocol 다이나믹하게 계속 사설 ip 분배해주는 하여 Web을 사용하도록 하는 protocol
	-- IP : 네트워크상에서 컴퓨터를 식별할 수 있는 주소
	-- DNS : IP주소를 인간이 쉽게 외우도록 맵핑한 문자열
	-- Port : IP주소가 컴퓨터를 식별할 수 있게 해준다면, Port번호는 해당컴퓨터의 구동되고 있는 프로그램을 구분할 수 있는 번호
2. HTML 이란?
    - 웹 문저를 기술하는 언어 , 웹문서를 표현하기 위해 태그들로 구성.
3. CSS 란 ?
    - HTML문서를 디자인적으로 예쁘게 만들어 정보 전달을 좀 더 효율적으로 하기 위해 만들어진 문서 
4. JavaScript (동적인 내용) > jQuery
    - (HTML, CSS,JavaScript)는 client
    - (PG는 Java나 Python, Node.js)는 서버단 : DB에 연결하고 상황에 따라 다르게 작동.
    - Ajax는 웹 페이지의 특정부분을 업데이트 서버와 데이터를 교환하는 기술.
5. 기본적인 웹 표준 문서의 상호관계
    - 웹표준은 웹문서 구조와 표현(디자인) , 동작을 별로도 구분
6. 웹 접근성을 높여주는 웹표준
    - 어떤 브라우저를 사용하던지. 일반인 또는 장애인 그리고 어떤 기기에서든지 모든사람이 해당 사이트를 제대로 볼 수 있는지를 의미.
    - 사이트가 웹표준을 준수하지 않을 경우, 다른 브라우저에서는 의도하지 않게 다른 컨텐츠가 표시되거나 화면에 표시가 되지 않을수 있다.
