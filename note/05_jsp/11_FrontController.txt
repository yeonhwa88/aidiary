url - pattern :
디렉터리 구조 : 디렉터리 형태로 서버의 해당 컴포넌트(서블릿)를 찾아서 실행하는 실행하는 구조. 
	       서블릿을 만들고 서블릿의 긴 주소가 보안상 문제가 되거나 너무 길어 xml이나 어노테이션을 이용해 매핑 할 때 사용했던 방식
확장자 구조 : 확장자 형태로 서버의 해당 컴포넌트를 찾아서 실행하는 구조 (*.do)
FrontController : 클라이언트의 다양한 요청을 한곳으로 집중시켜 개발 및 유지보수에 효율성 극대화
Command 구조 : 클라이언트로부터 받은 요청들에 대해서 서블릿이 작업을 직접 처리 하지 않고 해당 클래스가 처리하도록 한다.
	          FrontController 한군데서 처리하면 소스가 방대해져 큰 형태만 갖고 있고 세부적인 처리는 command에서 처리
